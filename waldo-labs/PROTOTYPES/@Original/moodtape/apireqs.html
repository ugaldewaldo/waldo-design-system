What Moodtape needs from the Brand Intelligence API

Every endpoint Moodtape reads — and where the gaps are

Moodtape is built on the Brand Intelligence API (bxcAPI). The brand-level sentiment is already there — distribution, trend, themes, category roll-ups, and a sentiment.drop alert event all ship today. What the API is missing is a securities layer: nothing maps a ticker to the portfolio of brands behind it, rolls those brands into a single security-level score, or exposes a market-wide index to anchor a landing page. And by design the bxcAPI has no stock-price data — so the price half of every chart is an explicit external join. The additions below add that thin securities envelope on top of the brand data that already exists.

Proposed additions

Five additions, all thin wrappers over data the API already has. The keystone is a /tickers/{symbol} resource that maps a security to its brand portfolio — every other addition derives from it. The one genuine outsider is stock price, which the bxcAPI should not own; we flag where the external join belongs.

GET /tickers/{symbol} ✗ Gap · keystone

The securities envelope the API is missing. Maps a ticker to the portfolio of brands behind it, with weights — so a security like PEP resolves to Pepsi, Lay's, Doritos, Gatorade and Quaker. Everything else (the rolled-up score, the tape, the movers, alerts) keys off this one resolution. Today there is no ticker/security concept anywhere in the bxcAPI.

{
  "symbol": "PEP",
  "company": "PepsiCo, Inc.",
  "sector": "Beverages & Food",
  "brands": [
    { "brand_id": "brand_pepsi",    "weight": 0.34 },
    { "brand_id": "brand_lays",     "weight": 0.22 },
    { "brand_id": "brand_doritos",  "weight": 0.18 },
    { "brand_id": "brand_gatorade", "weight": 0.16 },
    { "brand_id": "brand_quaker",   "weight": 0.10 }
  ]
}

Powers: search resolution, the brand-portfolio chips, and the inputs to every roll-up below.

GET /tickers/{symbol}/sentiment ✗ Gap

A single security-level sentiment object — the weighted roll-up of the portfolio's brand sentiment — plus distribution, a per-platform split, and a net_sentiment timeseries. Today the prototype fakes this by fanning out across /brands/{id}/mentions/summary for every brand and weighting client-side; this returns it in one call.

{
  "symbol": "PEP",
  "net_sentiment": 0.16,
  "wow_delta": -0.04,
  "mention_volume": 21400,
  "distribution": { "positive": 0.51, "neutral": 0.22, "mixed": 0.12, "negative": 0.15 },
  "by_platform": { "reddit": 0.08, "x": 0.21, "news": 0.04, "tiktok": 0.30, "youtube": 0.12 },
  "series": [{ "week": "2026-05-25", "net_sentiment": 0.16 }]
}

Powers: the big ticker score + badge, the ticker tape, the movers leaderboard, the sentiment-mix and by-platform panels.

GET /market/sentiment-index ✗ Gap

A single top-level read for the landing page: a market-wide sentiment index across the tracked universe, its week-over-week move, a by-sector breakdown, and a trend series. Today there's no aggregate above a single category, so the home view would have to fan out across every sector and ticker.

{
  "as_of": "2026-05-28",
  "index": 0.18,
  "wow_delta": 0.03,
  "by_sector": [
    { "sector": "Media & Entertainment", "net_sentiment": 0.21 },
    { "sector": "Restaurants & QSR",      "net_sentiment": -0.03 }
  ],
  "trend": [{ "week": "2026-05-25", "index": 0.18 }]
}

Powers: the four hero KPIs, the "sentiment by sector" bars, and the 52-week market-trend chart.

GET /tickers/{symbol}/price-sentiment ✗ Gap · external join

The overlay chart needs price and sentiment on the same timeline, plus markers where sentiment turned. The sentiment side comes from the roll-up above — but stock price is not, and should not be, the bxcAPI's job. The honest design is for the API to expose the sentiment series and the change annotations, and for the client (or a thin gateway) to join an external market-data feed for OHLC. We flag it here so the contract for the annotation/event side is explicit.

{
  "symbol": "PEP",
  "sentiment": [{ "week": "2026-05-25", "net_sentiment": 0.16 }],
  "events": [
    { "week": "2026-03-30", "direction": "down", "magnitude": 0.18, "label": "Snack recall chatter" }
  ],
  "price": "join externally — out of bxcAPI scope"
}

Powers: the dual-axis price-vs-sentiment chart and its event markers.

FIELD · ticker / sector alert scope ✗ Gap · small

The alert machinery is done — sentiment.drop and mention.spike events, preview/replay dry-runs, channels — it just speaks brands and categories. Letting scope accept a ticker or sector (resolved through /tickers/{symbol}) makes the alert form map 1:1 onto a security a trader actually watches.

{
  "trigger": {
    "event_type": "sentiment.drop",   // sentiment.drop | sentiment.spike | mention.spike
    "threshold": { "min_move": 0.20, "window": "4w" }
  },
  "scope": { "ticker": "TSLA" },        // or { "sector": "media" }
  "channels": [{ "type": "slack", "target": "#my-watchlist" }]
}

Powers: the alert-setup scope picker and the "would have fired N times" dry-run preview.
