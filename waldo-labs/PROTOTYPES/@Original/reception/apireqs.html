Ad social-listening · scoring & endpoint reference

How Breakthrough, Sentiment, and Pull-through are computed.

Reception measures the qualitative effectiveness of an ad campaign from the social conversation it generates — especially on TV/CTV, where there's no click to count. Every contributor below maps a raw signal to a 0–100 component, weighted into a pillar; the three pillars roll into one Reception Score. This page documents the equations, the /bxcapi endpoints and properties they require, whether each exists today, and the additions the API needs.

Status legend. ✓ Available = exists in /bxcapi today · ⚠ Partial = endpoint exists but a needed property/scope doesn't · ✗ Gap = MVP requirement, not in /bxcapi yet · ◷ V2 = deferred past MVP (no reliable source yet). The anchor already exists: GET /ads/{ad_id}/audience-reaction returns sentiment + verbatim themes from comments under a creative. The gaps are about user-owned campaigns, TV/CTV-airing→social linkage, campaign-level rollups, comment drill-in, and benchmarking.

Notation used in the equations

  • conv(c) — comments + posts reacting to campaign c in the flight window
  • Norm(x) = clamp(50 + 16.67·Z(x), 0, 100) — percentile-style 0–100 mapping
  • Z(x) — standardized vs the category competitive set over the window
  • decay_τ(d) = exp(−d / τ) — age decay in days from airing
  • on_topic — comments matched to the campaign's context spec
  • Σ w_i = 1 — weights sum to 1 per pillar
Breakthrough · attention

Breakthrough — did the ad cut through, or did it air to silence?

The hardest thing to measure on TV/CTV is whether anyone noticed. Breakthrough reads the volume and share of conversation a campaign generates relative to its own baseline and the category — not impressions you bought.

Breakthrough = clamp( 0.35·Norm(ConversationLift) + 0.30·Norm(ShareOfConversation) + 0.20·Norm(ConversationPerImpression) + 0.15·Norm(SpikeMagnitude), 0, 100 )
ConversationLift w = 0.35 mixed ⚠ Partial
raw = conv(campaign, window) / baseline_conv(brand, trailing_90d)

Sentiment · reception quality

Sentiment — how did people feel, and why?

Net positive/negative is the trunk, but the texture — funny, annoying, confusing, brand-love — is what tells a creative team whether the ad worked. The anchor endpoint already returns this; the gap is a standard reaction taxonomy so campaigns are comparable.

Sentiment = clamp( 0.40·Norm(NetSentiment) + 0.25·Norm(EmotionTexture) + 0.20·Norm(LoveVsBacklash) + 0.15·Norm⁻(ControversyIndex), 0, 100 )
NetSentiment w = 0.40 environmental ✓ Available
raw = sentiment.positive − sentiment.negative over campaign reactions

Pull-through · message landing

Pull-through — did the brand and message survive the joke?

A viral ad that people can't attribute to your brand is wasted reach. Pull-through measures whether commenters name the brand, repeat the intended message, and signal intent — the closest social proxy to "the creative did its job" when there's no conversion to track.

Pull-through = clamp( 0.40·Norm(BrandAttributionRate) + 0.35·Norm(MessageEcho) + 0.25·Norm(PurchaseIntentSignal), 0, 100 )
BrandAttributionRate w = 0.40 mixed ✗ Gap
raw = count(comments naming the brand) / count(on_topic comments) (vs naming only the celebrity, song, or meme)

Endpoint(s)

or comment-level brand-mention tags (proposal §4)

MessageEcho w = 0.35 mixed ⚠ Partial
raw = count(on_topic comments echoing the intended message/hook) / count(on_topic comments) — message from the campaign context spec

Proposed bxcAPI additions

What the API needs to ship Reception

Eight additions, in rough order of leverage. The first two — user-owned campaigns and TV/CTV airing→social linkage — are what make this product possible at all; the rest make it efficient.

§1 · highest leverage
User-owned campaigns — POST /campaigns, GET /campaigns/{id}

Today the API is brand/competitor-scoped (/brands/{id}/…). Reception needs a first-class object the user creates and owns — defined either by ad IDs (link mode) or a TV/CTV airing spec (no URL).

POST /campaigns
{
  "name": "Death to Plastic — Q2 CTV flight",
  "brand_id": "brand_liquiddeath_001",
  "type": "tv",                      // "link" | "tv"
  "ad_ids": [],                       // link mode
  "airings": [ /* see §2 */ ],         // tv mode
  "context": "Is the sustainability angle landing without reading preachy?",
  "match": { "terms": ["death to plastic", "murder your thirst"], "hashtags": ["#deathtoplastic"] }
}

Returns a campaign_id and an index-build status. GET /campaigns?brand_id=… lists a brand's campaigns (powers the campaign switcher and the vs-previous-campaigns table).

§2 · the TV/CTV unlock
Airing→social linkage — airings[] + match on a campaign

The whole reason TV/CTV is hard: there's no permalink. Let a campaign be defined by where and when it aired plus a match spec, and have the API attribute social conversation to that window.

{
  "airings": [
    { "start": "2026-05-01", "end": "2026-05-28",
      "networks": ["hulu", "roku", "comedy_central"],
      "dmas": ["new_york", "los_angeles"], "dayparts": ["primetime"] }
  ]
}

The API resolves the airing window + region and returns the matched conversation. Without this, a TV campaign can only be approximated with /discover/posts keyword search — no window scoping, no impressions tie-in.

§3
Campaign reception rollup — GET /campaigns/{id}/reception

/ads/{ad_id}/audience-reaction is per single ad and point-in-time. This rolls reactions across many ad IDs + organic posts + matched TV chatter into one object with a daily series (so the dashboard can show decay after airing).

{
  "campaign_id": "cmp_dtp_q2",
  "window": { "start": "2026-05-01", "end": "2026-05-28" },
  "comment_count": 38420,
  "sentiment": { "positive": 0.71, "neutral": 0.20, "negative": 0.09 },
  "attribution_rate": 0.88, "message_echo": 0.73,
  "by_platform": [ { "platform": "tiktok", "share": 0.30, "sentiment": {…} } ],
  "daily": [ { "date": "2026-05-01", "comments": 2140, "net_sentiment": 0.61, "airing": true } ]
}

§4
Comment-level feed — GET /campaigns/{id}/comments

The anchor endpoint returns one verbatim per theme — not enough for a verbatim wall or filtering. Paginate the actual comments with full metadata. (Equivalently: ?include=sample_content&sample_per_bucket=N on audience-reaction.)

{ "data": [
    { "platform": "youtube", "author": "StreamingDad",
      "text": "this came on during my show and I rewound it…",
      "sentiment": "positive", "theme": "saw it on Hulu",
      "engagement": 2100, "posted_at": "2026-05-26T01:10:00Z",
      "names_brand": true }
  ], "meta": { "next_cursor": "…" } }

§5
Canonical reaction taxonomy — ?taxonomy=v1 on audience-reaction

Free-form themes[] can't be compared across campaigns. Add a versioned, standard reaction taxonomy (funny, brand_love, mission_resonates, annoyed, confused, purchase_intent, hostile) alongside the free-form themes. Powers EmotionTexture, LoveVsBacklash, PurchaseIntentSignal and the cross-campaign Compare view.

§6
Brand attribution & message pull-through — attribution_rate, message_echo

The closest social proxy to "the creative worked" when there's no conversion. attribution_rate = share of comments naming the brand (vs only the celebrity/song); message_echo = share echoing the campaign's declared message. The Martha Stewart example in the demo scores 92 on breakthrough but only 55 on pull-through precisely because these were low — a distinction nothing in the API surfaces today.

§7
Reception benchmarking — GET /category/{id}/reception-benchmark

Percentile a campaign's pillars against the category set and against the brand's own prior campaigns (?compare=category|prior_campaigns). Brand-dataset benchmarks exist, but nothing benchmarks ad reaction. Powers the Compare tab's percentile bars and the median markers.

§8
TV impressions tie-in — impressions on airings[]

To compute conversation-per-impression (the CTV efficiency proxy that replaces CTR), attach impressions per airing from a TV-measurement partner (iSpot, VideoAmp, Nielsen). Deferred to V2 until a source is locked, but the field shape should be reserved on the airing object now.
