KPI Stat Card with Trend

A card that displays a large metric value with a label and a contextual trend indicator below. Different from a plain card — it's purpose-built for dashboard KPIs where the direction of change matters as much as the number itself.

Differs from plain Card: Has a fixed 3-zone layout — label → big value → trend row. The trend badge encodes direction: up = red (pressure rising), down = teal (easing), flat = muted. The top border stripe uses a subtle teal gradient to anchor it visually as a metric surface.

Tokens used: --surface bg, --radar stripe + down badge, --red up badge, --text-3 label, --text value.
Prop:trend: "up" | "down" | "flat"
Prop:value: string
Prop:label: string
Prop:trendText: string

Horizontal Progress Bar (inline label)

A single-row layout with a category name on the left, a filled progress bar in the middle, and a percentage value on the right. Used for ranking and comparison lists. Distinct from the DS Bar Chart — no axes, no chart container, designed to stack as a list inside a card.

Leaderboard Row

A list row for ranked "top N" lists. Each row carries: rank number + brand name + discount-type badge + offer copy + depth pill on the right. More information-dense than a standard list item — it combines three data types (rank, metadata badge, numeric indicator) in one scannable row.

Differs from a plain list row: Three visual zones in one row — rank (mono, fixed width), content block (brand + badge + offer text), depth pill (right-aligned, gold opacity-encoded to convey severity at a glance). Ranks 1–3 get gold color treatment; rank 4+ are muted. The depth pill opacity scales with depth: depth ≥ 60% = full opacity, 30% = ~0.59.

Tokens used: --surface-3 badge bg, --text-3 rank, --gold depth pill + top ranks, --text-2 offer copy.
Prop:rank: number
Prop:brand: string
Prop:badgeLabel: string
Prop:offerCopy: string
Prop:depth: number (0–100)

Animated Brand Mark / Logo Badge

A circular SVG radar icon with a rotating sweep arm and optional ripple/pulse animations. Used as the app identity mark in the topbar — it communicates "live data, always scanning." Distinct from a static icon because the animation is load-bearing: it signals active monitoring state.

Why this is a component, not an icon: The animation is semantically meaningful — it signals "actively scanning / live data." A static version would lose that meaning. The sweep rotation speed (3s), ripple timing, and opacity values are all intentional and should be DS-standardized so every app that signals live intelligence uses the same cadence.

Tokens used: --radar / currentColor for all strokes and fills. Container badge uses rgba(--radar-rgb, 0.1) bg and rgba(--radar-rgb, 0.25) border.

Animation tokens to define: --anim-radar-sweep: 3s linear infinite, --anim-radar-ripple: 3s ease-out infinite.
Prop:size: "sm" | "md" | "lg" | "xl"
Prop:ripple: boolean
Prop:contained: boolean
Prop:speed: number (seconds, default 3)
