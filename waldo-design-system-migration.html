Migration Guide

Old Waldo DS → new tokenized DS. Use this to walk through your Figma file (and code) replacing legacy styles and variables with the new tokens. Sections cover text styles, solid fills, translucent fills, text colors, lines and accents.

Last updated · May 2026

Solid fills (backgrounds)

Old FILL/* styles → new semantic surface tokens (which reference the zinc primitive scale).

Old styleMaps to primitiveSemantic tokenUse case
FILL/950 🌙 DARK→zinc-950→surface-defaultMain app background
FILL/900 🌙 REGULAR→zinc-900→surface-elevatedCards, panels
FILL/850 🌙 HIGH→zinc-850→surface-elevated (alt)Sidebar, raised panels
FILL/800 🌙 LOW→zinc-800→surface-sunkenInputs, code blocks
FILL/750→zinc-750→—Custom step, rarely used as surface directly
FILL/700 🌙 REGULAR→zinc-700→surface-popoverPopovers, dropdowns
FILL/600 🌙 MENU→zinc-600→accentMenu items, accent panels
FILL/500 🌙 HIGHEST→zinc-600 (visually highest depth)→accentTopmost solid layer
Note: in the old system, "500 HIGHEST" meant the highest visual depth (lightest in dark mode), not the highest in a color scale. In the new system, primitives follow the standard Tailwind scale (50 = lightest, 950 = darkest) and the "depth" is encoded in the semantic token name, not the primitive number.

Translucent fills (FILL OPACITY)

Old FILL OPACITY/* styles → new alpha-primitive backed semantic tokens. These are the colors that need to "adapt" when sitting over images or other content.

Old styleMaps to alpha primitiveSemantic tokenUse case
FILL OPACITY/Widget BG→zinc-950 / 40%→surface-frostedTranslucent widgets over hero images
FILL OPACITY/Widget BG Hover→zinc-950 / 50%→surface-frosted-hoverHover of the above
FILL OPACITY/800 🌙 LOW→zinc-200 / 5%→interactive-hoverVery subtle hover overlay on any surface
FILL OPACITY/700 🌙 REGULAR→zinc-200 / 10%→interactive-pressedActive / pressed state
FILL OPACITY/500 🌙 HIGHEST→zinc-200 / 15% (closest)→interactive-pressed (or custom)Strongest hover, may need a new semantic name
Why this matters: in the old model you had to choose between FILL/700 REGULAR (solid) and FILL OPACITY/700 REGULAR (translucent) based on what was behind. In the new model, components just use the semantic name (surface-frosted) and the right behavior emerges from the token definition.

Accent & brand

Old ACCENT/* and HEYKID/* styles → consolidated brand colors (marketing only) + product accents.

Product UI accents (separate from marketing brand)
(brand primary)→accent-brandPrimary CTA in product UI
(warning)→accent-warningWarning states
(destructive)→accent-destructiveDestructive actions, errors
Important distinction: brand-* (5 families × 5 steps, marketing scale) is different from accent-* (semantic product UI accents). Marketing pages use brand-pink-500 for a flashy CTA; product UI uses accent-brand (= teal-500) for the primary button.
